Deployment Guide

Deploy HMG SermonScribe v2 to the web in minutes. No server. No database. No monthly bill. Just a modern static host and a HTTPS URL.

Before You Begin

Important: The Web Speech API requires a secure context (HTTPS) and microphone permission. Localhost is allowed during development, but production must be served over HTTPS. All platforms listed below provide HTTPS automatically.
  • A modern code editor (VS Code, Notepad++, or Acode on Android).
  • A GitHub account (free) if you want version control and automated deployment.
  • The sermon v2 folder containing all application files.

Step 2 — Choose a Platform

Recommended Free Platforms
  • GitHub Pages — Best for version control and open-source visibility. Automatic HTTPS. Drag-and-drop or Git-based. Custom domain supported.
  • Cloudflare Pages — Fastest global CDN. Excellent for Nigerian users on slow networks. Connects to GitHub directly. HTTPS and custom domains free.
  • Vercel — Zero-config deployment from GitHub. Fast previews. Great if you plan to scale or add serverless functions later. HTTPS default.
  • Netlify — Drag-and-drop deploy. Form handling if you later add a contact page. Branch previews. Free HTTPS and custom domain.
  • Static Host / cPanel — If your church already has a web hosting account, upload the files via File Manager or FTP. Ensure the host forces HTTPS.

Step 3 — Deploy to GitHub Pages (Recommended)

3.1 Create a GitHub repository
  1. Go to github.com/new and create a new public repository (e.g., hmg-sermonscribe-v2).
  2. Do not initialise it with a README or licence if you already have local files.
3.2 Upload the sermon v2 folder

Open your terminal in the folder that contains the sermon v2 folder:

git init
git add "sermon v2/"
git commit -m "Initial commit: HMG SermonScribe v2.0"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/hmg-sermonscribe-v2.git
git push -u origin main

Replace YOUR_USERNAME with your actual GitHub username.

3.3 Enable GitHub Pages
  1. On your GitHub repository page, go to Settings → Pages (in the left sidebar).
  2. Under Build and deployment, select Source: Deploy from a branch.
  3. Choose the main branch and the / (root) folder, then click Save.
  4. Wait 1–2 minutes. GitHub will show a green badge with your live URL.
3.4 (Optional) Use GitHub Actions for automation

The included .github/workflows/deploy.yml file will automatically redeploy every time you push to the main branch. In Pages settings, select Source: GitHub Actions instead of branch.

Step 4 — Deploy to Cloudflare Pages (Best for Speed)

  1. Sign up at dash.cloudflare.com.
  2. In the dashboard, go to Pages → Create a project.
  3. Connect to your GitHub repository hmg-sermonscribe-v2.
  4. Choose the main branch.
  5. Set the build command to nothing (leave blank). Set the build output directory to /sermon v2 if your repo root is the parent folder. If your repo root is the app contents directly, use /.
  6. Click Save and Deploy.

Step 5 — Deploy to Vercel

  1. Go to vercel.com/new.
  2. Import hmg-sermonscribe-v2 from GitHub.
  3. Vercel will auto-detect it as a static site. Set the Root Directory to sermon v2 if applicable.
  4. Click Deploy. Live in under 30 seconds.

Step 6 — Deploy to Netlify

  1. Go to app.netlify.com/drop.
  2. Zip the sermon v2 folder on your computer.
  3. Drag the ZIP file onto the Netlify page.
  4. Netlify will instantly deploy it and give you a random URL.

Step 7 — Post-Deployment Checklist

  1. Open the live URL in Chrome or Edge on Android or desktop.
  2. Click Allow when the browser asks for microphone permission.
  3. Click Start Recording and speak a sentence. Text should appear within 2 seconds.
  4. Click Stop and verify the text stays in the editor.
  5. Refresh the page and confirm the sermon is still in the sidebar list (IndexedDB persistence).
  6. Click Export → Print and verify the print preview looks clean.
  7. Click the browser menu and select Install or Add to Home Screen. The PWA should install with the HMG S icon.
  8. Turn off Wi-Fi / mobile data and reload the app. The interface should still appear (cached by Service Worker).
  9. Visit live.html and confirm it opens a black screen for projector captions.
  10. Visit analytics.html and confirm the dashboard loads (it will show zero stats until you have sermons).
